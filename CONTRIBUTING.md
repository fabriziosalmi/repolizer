# Contribuire a Repolizer

Grazie per il tuo interesse a contribuire a Repolizer! Questo documento fornisce linee guida per contribuire al progetto.

## Come Contribuire

### Segnalazione di Bug

Se hai trovato un bug, per favore crea una issue con i seguenti dettagli:

- Una descrizione chiara e concisa del bug
- Passaggi per riprodurre il problema
- Comportamento atteso
- Screenshot (se applicabile)
- Informazioni sul tuo ambiente (sistema operativo, versione di Python, ecc.)

### Suggerimenti di Miglioramento

Se hai idee per migliorare Repolizer, crea una issue con i seguenti dettagli:

- Una descrizione chiara e concisa del miglioramento proposto
- Motivazione: perché questo miglioramento sarebbe utile?
- Eventuali implementazioni alternative considerate

### Pull Request

1. Fork del repository
2. Crea un branch per la tua feature (`git checkout -b feature/nome-feature`)
3. Implementa le tue modifiche
4. Aggiungi test per le nuove funzionalità
5. Assicurati che tutti i test passino
6. Commit delle modifiche (`git commit -m 'Aggiungi nuova feature'`)
7. Push al branch (`git push origin feature/nome-feature`)
8. Apri una Pull Request

## Stile del Codice

- Segui le convenzioni PEP 8 per il codice Python
- Usa docstring in formato Google per documentare funzioni e classi
- Mantieni il codice semplice e leggibile
- Aggiungi commenti dove necessario per spiegare logiche complesse

## Test

- Aggiungi test unitari per le nuove funzionalità
- Assicurati che tutti i test passino prima di inviare una Pull Request
- Esegui i test con `pytest`

## Documentazione

- Aggiorna la documentazione per riflettere le modifiche apportate
- Usa un linguaggio chiaro e conciso
- Includi esempi dove appropriato

## Processo di Review

- Le Pull Request saranno revisionate dai maintainer del progetto
- Potrebbero essere richieste modifiche prima dell'accettazione
- Una volta approvate, le modifiche verranno unite al branch principale

## Licenza

Contribuendo a Repolizer, accetti che il tuo codice sarà rilasciato sotto la licenza MIT del progetto.
